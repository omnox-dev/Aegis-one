\section{Introduction}
This document outlines the architecture, database design, and file structure of the application. The system is a full-stack web application designed to manage university activities, grievances, internships, and resources.

\section{System Architecture}
The system follows a modern client-server architecture:
\begin{itemize}
    \item \textbf{Frontend}: Built with Next.js (React), providing a responsive user interface.
    \item \textbf{Backend}: Powered by FastAPI (Python), handling API requests and business logic.
    \item \textbf{Database}: Relational database (PostgreSQL/SQLAlchemy) for data persistence.
\end{itemize}

\section{Project File Structure}
The following is a summary of the key files and directories in the project.

\subsection{Backend Structure (Python/FastAPI)}
\begin{itemize}
    \item \texttt{backend/app/models/}
    \begin{itemize}
        \item \texttt{user.py}: User authentication and profile model.
        \item \texttt{course.py}: Course and enrollment data.
        \item \texttt{grievance.py}: Grievance submission and tracking.
        \item \texttt{club.py}: Club management and events.
        \item \texttt{internship.py}: Internship listings and applications.
        \item \texttt{forum.py}: Discussion forum posts and comments.
        \item \texttt{resource.py}: Academic resources upload/download.
        \item \texttt{incident.py}: Emergency/Safety incident reports.
    \end{itemize}
    \item \texttt{backend/app/api/}
    \begin{itemize}
        \item \texttt{auth.py}: Authentication endpoints (login/register).
        \item \texttt{users.py}: User management endpoints.
        \item \texttt{dashboard.py}: Aggregated data for dashboards.
        \item \texttt{grievances.py}, \texttt{clubs.py}, \texttt{courses.py}, etc.
    \end{itemize}
\end{itemize}

\subsection{Frontend Structure (Next.js)}
\begin{itemize}
    \item \texttt{frontend/src/app/}
    \begin{itemize}
        \item \texttt{layout.tsx}: Main application layout (navbar, footer).
        \item \texttt{page.tsx}: Landing page.
        \item \texttt{dashboard/}: User and Admin dashboards.
        \item \texttt{login/}: Authentication pages.
    \end{itemize}
    \item \texttt{frontend/src/components/}
    \begin{itemize}
        \item \texttt{dashboards/}: Dashboard widgets and views.
        \item \texttt{AuthSplashScreen.tsx}: Login splash component.
    \end{itemize}
\end{itemize}
